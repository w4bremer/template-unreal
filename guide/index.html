# Overview

This guide will explain how to use the generated plugin code. Which features can be enabled and what they generate.

# Get started

This template generates plugin code for Unreal Engine (opens new window) projects. In order to successfully compile and use the plugin, you need to have a version of the Unreal Engine installed. The engine can be downloaded from here (opens new window). Furthermore, we assume a basic understanding of Unreal Engine projects. If you have questions how the Engine works you can checkout the documentation (opens new window) or the community (opens new window). You should be familiar how Unreal Blueprints work to use the plugin code. However, you do not need to write UE C++ code unless you want to modify the template.

If you did not receive already generated code you can simply generate your own. Please follow the documentation for the ApiGear CLI (opens new window) or the ApiGear Studio (opens new window) for more information about the general code generation.

TIP

For questions regarding this template please go to our discussions page (opens new window). For feature requests or bug reports please use the issue tracker (opens new window).

# Features

This template supports a series of different features. These can be basically split into two groups, core and extended.

  • The full set of core features generates a view model for the api. This can be used to implement a working C++ or Blueprint service and directly use it in your UI project.
  • The extended features can be used in combination with api and add more functionality on top.

Each feature can be selected using the solution file or via command line tool.

Please note: features are case sensitive, make sure to always use lower-case.

# Core

  • api - will only generate pure header files and a basic implementation for data types
  • stubs - adds a basic stubs for the api, but without any plugin setup
  • scaffold - generates plugin code for UE projects. This is the first state which can be compiled.

# Extended

  • simulation - this provides a backend which can be simulated using the CLI (opens new window) or the Studio (opens new window)
  • olink - this provides a backend which can be connect to any of the other technology templates with support for ObjectLink (opens new window)
  • monitor - this generates a middleware layer which logs all API events to the CLI (opens new window) or the Studio (opens new window)

There is also an internal feature apigear. This will only generate the connectivity plugin for the extended features.

TIP

The meta feature all enables all specified features of the template. If you want to see the full extent of the generated code all is easiest solution. Please note, all is part of the code generator and not explicitly used within templates.
