Quick-Start

This document only explains the bare minimum needed to know to get from an API description to a functional Unreal Engine plugin.

1. Install the code generatorâ€‹

Get the ApiGear Studio or ApiGear CLI. For more information check the ApiGear documentation.

2. Get the templateâ€‹

There are several options to get the template. Installation via the Studio or the CLI. Alternatively it is possible to clone, to download from github.

Installation via CLIâ€‹

When using the CLI only the highlighted line is imported. You can always check whether installation was successful via the template list command.

$ apigear template list
list of templates from registry and cache
name                       | installed | registry | url
apigear-io/template-unreal | false     | true     | https://github.com/apigear-io/template-unreal.git
...
$ apigear template install apigear-io/template-unreal
$ apigear template list
list of templates from registry and cache
name                       | installed | registry | url
apigear-io/template-unreal | true      | true     | https://github.com/apigear-io/template-unreal.git
...

Installation via Studioâ€‹

From within the studio the installation is really simple.

  1. Open an existing project or create an new one
  2. Go to the Templates tab
  3. Click Install on the apigear-io/template-unreal entry

Clone from githubâ€‹

In case you want to check or modify the source code of the template, it is easier to clone or download the repository.

$ git clone https://github.com/apigear-io/template-unreal.git

3. Set up projectâ€‹

For a project we usually need two files. The solution which specifies what APIs and which template to use for it. And at least one API module file. Bot should ideally be in a folder called apigear next to each other.

Alternatively, you can also use the Studio to create a new project and modify the two example files.

Solution fileâ€‹

Create a solution file. The example below specifies

  • the helloworld.module.yaml which contains the API in line 8
  • the output directory in line 9
  • the apigear-ui/template-unreal to be used in line 10. This can also be a path to a local copy of the template.
  • enables the stubs feature of the template, a simple implementation in line 13. For all available features check the overview.
helloworld.solution.yaml
schema: "apigear.solution/1.0"
name: ue_docs
version: "0.1"

layers:
  - name: ue_docs
    inputs:
      - helloworld.module.yaml
    output: ../ue_docs
    template: apigear-io/template-unreal
    force: true
    features:
      - stubs

API module fileâ€‹

Create the helloworld.module.yaml with the example content:

helloworld.module.yaml
schema: apigear.module/1.0
name: io.world
version: "1.0"

interfaces:
  - name: Hello
    properties:
      - { name: last, type: Message }
    operations:
      - name: say
        params:
          - { name: msg, type: Message }
          - { name: when, type: When }
        return:
          type: int
    signals:
      - name: justSaid
        params:
          - { name: msg, type: Message }
enums:
  - name: When
    members:
      - { name: Now, value: 0 }
      - { name: Soon, value: 1 }
      - { name: Never, value: 2 }
structs:
  - name: Message
    fields:
      - { name: content, type: string }

Generate via CLIâ€‹

The following snippet shows how the CLI can be run.

$ apigear generate solution apigear/helloworld.solution.yaml 
17:52:20 INF generated 92 files in 30ms. (33 write, 0 skip, 59 copy) topic=gen
  • generate tells the CLI that it should generate code
  • solution specifies that we want to run a solution file

Generate via Studioâ€‹

  1. Open the project
  2. Go to the Solutions tab
  3. Click Run on the helloworld.solution.yaml entry

5. Use the plugin in your Unreal Engine projectâ€‹

The generated code supports Blueprint and C++ implementations. The following paragraphs show it can be used. Make sure to have the generated plugin code in your projects plugins directory as noted in the previous section.

Blueprintâ€‹

In Blueprint we first create an object of the type Io World Hello. This can then be directly used or stored in a variable. In the screenshot we use the asynchronous say function to say "Hello world".

C++â€‹

In order to use the plugin code outside of its module we have to add IoWorld to the list of dependency modules. This can be done like this:

ue_docs.Build.cs
PublicDependencyModuleNames.AddRange(new string[] { "Core", "CoreUObject", "Engine", "InputCore", "IoWorld" });

Afterwards, we can simply include the header files for the api interface, the stub implementation and use it. The following code is an example and should not be used like this in production code.

ue_docsGameModeBase.cpp
#include "ue_docsGameModeBase.h"
#include "Implementation/IoWorldHello.h"
#include "Generated/api/IoWorld_data.h"
#include "Generated/api/IoWorldHelloInterface.h"

void Aue_docsGameModeBase::InitGame(const FString& MapName, const FString& Options, FString& ErrorMessage)
{
    Super::InitGame(MapName,Options, ErrorMessage);

    // Note: UObjects should be referenced from an Unreal type or property, otherwise they might be removed by garbage collection
    IIoWorldHelloInterface* HelloWorldObjPtr = NewObject<UIoWorldHello>();
    FIoWorldMessage MyMsg;
    MyMsg.content = FString("Hello world");
    HelloWorldObjPtr->Say(MyMsg, EIoWorldWhen::NOW);
}
caution

UObjects should be referenced from an Unreal type or property, otherwise they might be removed by garbage collection. For more information check the object handling documentation.
